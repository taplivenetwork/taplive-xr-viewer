# Governance

This project is governed by the Core Team, responsible for technical direction, release management, and community oversight.

## Roles

### Maintainers (@RepoAdmin)
- **Responsibilities**: Merge PRs, manage releases, enforce code of conduct.
- **Access**: Admin rights to the repository.

### Code Owners (@TechLead, @SrDev)
- **Responsibilities**: Review code changes in `Assets/Scripts/`, ensure architectural integrity.
- **Access**: Write access, requested reviewers.

### Contributors
- **Responsibilities**: Submit PRs, report bugs, suggest features.

## Decision Making
Major technical decisions are proposed via RFC (Request for Comments) issues and approved by a majority of Maintainers. 

## Code of Conduct
All participants are expected to adhere to the standard Contributor Covenant Code of Conduct.
